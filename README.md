# XHS Keyword Explorer

> 小红书关键词挖掘与运营分析工具。基于 OpenCLI + 浏览器自动化，批量采集搜索下拉框关联词，结合笔记互动数据分析，筛选出高互动、低竞争的优质关键词。

---

## 项目简介

小红书的内容分发很大程度上依赖搜索流量。本工具帮助运营人员：

1. **种子词扩展**：输入少量种子关键词，自动采集搜索下拉框中的关联词
2. **数据批量采集**：批量搜索每个关键词，收集 Top 50 笔记的互动数据
3. **指标计算**：自动计算竞争度、互动度、优质度评分
4. **策略输出**：基于数据给出内容投放建议

## 快速开始

### Step 1: 准备种子关键词

编辑 `examples/keywords.txt`，每行一个关键词：

```
苏州酒馆
苏州酒吧
苏州清吧
苏州精酿
学士街
苏州小酒馆
苏州夜生活
```

### Step 2: 采集下拉框关联词

```bash
# 先确认浏览器已在小红书页面
opencli browser <your-profile> state | grep xiaohongshu

# 运行采集脚本
PROFILE=<your-profile> bash scripts/batch_suggest.sh examples/keywords.txt /tmp/xhs_screenshots
```

脚本会逐个输入关键词，等待下拉框加载，截图保存。之后人工读取截图即可提取关联词。

### Step 3: 批量搜索笔记数据

```bash
opencli xiaohongshu search "苏州酒馆" --limit 50 -f json > data/苏州酒馆.json
```

对所有关键词重复执行，收集 JSON 数据。

### Step 4: 数据导入飞书

```bash
lark-cli auth login --no-wait --json
# 按提示完成 OAuth

# 创建表格并导入数据（详见 docs/workflow.md）
```

## 核心脚本说明

### `scripts/batch_suggest.sh`

**功能**：在小红书搜索框批量输入关键词，自动截图保存下拉框建议。

**用法**：
```bash
PROFILE=p9wmr42g KEYWORDS_FILE=examples/keywords.txt OUTPUT_DIR=/tmp/xhs_screenshots bash scripts/batch_suggest.sh
```

**环境变量**：

| 变量 | 必填 | 默认值 | 说明 |
|---|---|---|---|
| `PROFILE` | 是 | - | OpenCLI browser profile ID |
| `KEYWORDS_FILE` | 是 | - | 关键词列表文件，每行一个 |
| `OUTPUT_DIR` | 否 | `/tmp/xhs_screenshots` | 截图保存目录 |
| `WAIT_AFTER_TYPE` | 否 | `5` | 输入后等待秒数（下拉框 API 有延迟） |
| `WAIT_BETWEEN` | 否 | `5` | 关键词间间隔秒数 |

**防检测机制**：
- 随机延迟（模拟人类操作间隔）
- 输入前恢复页面、输入后透明化笔记卡片再截图
- 不刷新页面（避免元素引用失效）

**已知限制**：
- 小红书搜索建议有缓存延迟，截图中的建议偶尔是上一个词的缓存
- 即使显示缓存，这些词本身也是真实被搜索过的关联词，仍有参考价值

## 数据分析标准

### 竞争度（基于搜索结果笔记总数）

| 级别 | 范围 |
|---|---|
| 低竞争 | < 1,000 篇 |
| 中竞争 | 1,000 - 10,000 篇 |
| 高竞争 | > 10,000 篇 |

### 互动度（基于 Top 50 笔记平均点赞+收藏）

| 级别 | 范围 |
|---|---|
| 高互动 | > 5,000 |
| 中互动 | 1,000 - 5,000 |
| 低互动 | < 1,000 |

### 优质度评分矩阵

| 互动度 \ 竞争度 | 低竞争 | 中竞争 | 高竞争 |
|---|---|---|---|
| 高互动 | **强烈推荐** | 推荐 | 可选 |
| 中互动 | **推荐** | 可选 | 备选 |
| 低互动 | 备选 | 不推荐 | 不推荐 |

**策略**：优先布局"高互动 + 低竞争"和"中互动 + 低竞争"的关键词。

## 坑点记录

1. **不要执行 `opencli browser <profile> open "https://www.xiaohongshu.com"`**  
   这会刷新页面，导致 `ref=2` 等 DOM 元素引用失效。

2. **小红书搜索建议有缓存延迟**  
   输入关键词后，下拉框可能先显示历史记录或上一个词的缓存。建议等待 5-8 秒再截图。

3. **下拉框被笔记卡片遮挡**  
   通过 JS 将笔记卡片 `opacity: 0.1` + `pointer-events: none` 解决。

4. **lark-cli 需要 OAuth**  
   运行 `lark-cli auth login --no-wait --json`，在浏览器完成登录后脚本自动轮询。

5. **某些缩写词被误识别**  
   例如 "IPA"（精酿啤酒）在小红书被识别为 "iPad"，下拉框显示的是电子产品相关内容。
